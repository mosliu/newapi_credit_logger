# NewAPI Key 余额监控系统开发计划

## 1. 项目目标

构建一个可配置、可追踪、可扩展的余额监控系统，支持：

- 管理后台配置多个 NewAPI Key 与地址。
- 按配置周期（默认 60 秒）定时拉取余额并记录。
- 前端页面查询监控对象与余额历史记录。
- 数据库同时支持 SQLite、MySQL、PostgreSQL。
- 日志按文件落盘，满足运维排障。
- 支持 Docker 部署与分发（镜像构建、启动编排、版本发布说明）。

## 2. 已确认需求范围

- 监控源配置：`base_url`、`api_key`、`interval_seconds`、`enabled`。
- 监控源说明字段：`key_owner`、`remark`。
- 定时采集：每 1 分钟可配置。
- 记录结果：成功/失败都要入库。
- 查询展示：列表 + 历史趋势 + 失败记录。
- 日志：使用 `loguru`，记录到项目日志目录。
- 交付 Docker 部署与分发能力：`Dockerfile`、`docker-compose`、镜像使用文档。

## 3. 技术方案

- Web/API：FastAPI
- ORM/迁移：SQLAlchemy 2.x + Alembic
- 调度：APScheduler
- HTTP 客户端：httpx
- 日志：loguru
- 前端：服务端渲染管理页（后续可拆前后端）
- 包管理：uv
